# 审批中心指引

这篇文档是产品帮助内容的示例页，主要用来验证目录结构、阅读体验和搜索索引字段。

## 推荐的内容组织方式

产品功能说明尽量按照用户任务来写，而不是按页面菜单生硬拆分：

- 这个功能解决什么问题
- 谁会使用它
- 在哪里进入
- 如何完成关键操作
- 常见异常和排查路径

## 搜索元数据建议

为了让后续 `Typesense` 搜索更准确，建议在文档层面补充：

- 功能别名
- 角色标签
- 部署类型
- 适用版本

这些字段既可以写在 frontmatter，也可以在索引脚本里做额外映射。
